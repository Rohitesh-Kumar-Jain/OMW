OMW Wordnet Documentation: Variants

This page provides documentation for adding variants to senses for the Open Multilingual Wordnet.

One of the extensions we have is allowing different variants of a lemma. These can be described by attribute value pairs. A common use for these is adding transcriptions for non-ascii lemmas. 

        <LexicalEntry id="w1">
            <Lemma writtenForm="doog" partOfSpeech="n"/>
	    <Form writtenForm="doogen">
	      <Tag category="morph">pl</Tag>
	    </Form>
	    <Form writtenForm="döög">
	      <Tag category="geo">Germany</Tag>
	    </Form>
	    <Form writtenForm="döögen">
	      <Tag category="morph">pl</Tag>
	      <Tag category="geo">UK</Tag>
	    </Form>
	    <Form writtenForm="dooog">
	    </Form>
	    <Sense id="test-dog-n-1" synset="test-dog-n">
            </Sense>
        </LexicalEntry>

Developers' Notes

The attributes and values are entered when the database is first created using scripts/load-form-meta.py and are stored in the tables fmt (attributes) and fml (values) respectively.

There is a sample wordnet to test this in /omw/tests/omw_doc_variants.html.
